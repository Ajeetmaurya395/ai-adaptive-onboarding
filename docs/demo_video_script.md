# Demo Video Script

## Goal

Deliver a 2–3 minute walkthrough that shows the adaptive onboarding value clearly and quickly.

## Suggested Flow

### 1. Problem Framing

Open on the `Hackathon` page.

Say:

> Traditional onboarding is static. Our system adapts training to what a new hire already knows and what the target role still requires.

### 2. Input Stage

Open `Upload`.

Choose one sample scenario, for example `Cloud Engineer`, and explain that the app also includes analytics and operational scenarios to show cross-domain scalability.

Say:

> We can either upload real inputs or instantly demonstrate the flow using domain-specific samples.

### 3. Skill Gap Analysis

Run the analysis and open `Analysis`.

Show:

- fit gauge
- skill treemap
- category coverage
- skill explorer

Say:

> The engine parses the resume and job description, normalizes the extracted skills, and identifies exactly what is already covered versus what is still missing.

### 4. Adaptive Roadmap

Open `Roadmap`.

Show:

- prioritized roadmap steps
- documentation
- free video
- paid course
- book / study material
- practice resource

Say:

> Instead of assigning the same curriculum to everyone, the engine creates a targeted learning plan only for the missing skills.

### 5. Explainability

Open `Reasoning`.

Say:

> Judges asked for reasoning trace, so the system explains why each skill gap matters and how the recommendation was produced.

### 6. Assistant

Open `Assistant`.

Ask:

> What should this candidate do first to improve the score fastest?

Say:

> The Qwen assistant lets users ask follow-up questions directly on top of the analysis and roadmap context.

### 7. Reliability

Open `Evaluation`.

Say:

> We also benchmark extraction quality with expected outputs and expose accuracy, precision, recall, and F1.

### 8. Close

End on `Hackathon`.

Say:

> This gives organizations a grounded, explainable, and adaptive onboarding engine that reduces redundant training while staying aligned to role-specific competency.
